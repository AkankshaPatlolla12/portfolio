Ma.ai


DESCRIPTION

Deliverables include: (1) an empathy → define → ideate → iterate decision log; (2) a focused problem statement with design principles (trust, 24/7 accessibility, context-awareness); (3) a working Streamlit prototype; and (4) a comparative evaluation across Gemini, Copilot, and ChatGPT to derive quality criteria (warmth, thoroughness, clarity + safety).

OBJECTIVE

Showcase the ability to translate user pain points into a narrowly scoped, medically cautious chatbot concept—and to prototype and evaluate it quickly against concrete, audience- relevant criteria.

PROCESS

We empathized and defined the problem by capturing parental stress signals and the risks of generic LLM advice; set design principles—medically accurate trust, accessibility, empathy with expertise, and age-/stage context; and ideated a guided intake (“How old is your baby and what’s worrying you?”), an immediate crisis mode, curated topic paths, and a parent mood/habit tracker.

TOOLS & TECHNOLOGIES USED

Built with Python and a lightweight SQLite (sqlite3) database, integrated the Google Gemini API for LLM capabilities, and deployed on Streamlit for a fast, shareable interactive prototype.

VALUE PROPOSITION

Ma.aI gives new parents fast, trustworthy, age-aware guidance - prioritizing medically cautious next steps over generic chatbot advice.
